3) Pricing Enforcement

For each pricing_model:

unit_fixed:

Use Price_AED as is; it must be non-null and > 0.

If 0.00, look up exact Code in products_view (from the CSVs) and correct it; if still missing, flag requires_fix=true.

unit_by_height:

Group by {Product_Family, OD_mm} and vary by Height_mm.

Each row represents a priced height variant. Price must be present. If not, flag requires_fix=true.

unit_by_length:

One row per length (Length_mm required).

Price must be present. If not, flag requires_fix=true.

unit_by_lw_fixed_h:

One row per Length_mm × Width_mm.

Price must be present; if missing, flag requires_fix=true.

linear_meter:

The selling unit is meter. Create fields:

rate_aed_per_m (nullable until the user supplies rates)

min_length_m (default 1.0 unless the family specifies a minimum)

A quoted line computes: line_total = rate_aed_per_m * requested_length_m * quantity.

If rate_aed_per_m is null → block quotation for that family and return a structured prompt asking for the rate(s).

Also: if Price_AED == 0 for any non-linear item, set requires_fix=true and include it in a “Fix List” report.

4) Dedupe + Canonical Names

5) Proposal Math

For quote building:

unit_fixed: line_total = Price_AED * quantity

unit_by_height: line_total = Price_AED * quantity (user picks height variant)

unit_by_length: line_total = Price_AED * quantity (user picks length variant)

unit_by_lw_fixed_h: line_total = Price_AED * quantity (user picks size)

linear_meter: line_total = rate_aed_per_m * requested_length_m * quantity

Always include image_url and joule_j on the line if present.
